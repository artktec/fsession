FreeSWITCH JavaScript zen with fSession

fSession is a small JavaScript library which attempts to help in quickly developing FreeSWITCH applications in JavaScript based around sessions. This is inline with how FreeSWITCH handles calls and conferencing. Hopefully it will help out those people coming from a web development background a lot. fSession also has nice things like Time parsing so it's quite easy to change things from minutes to milliseconds and back. It also has all of the built in constants as expected. With an API so that it's easy to add your own environment constants as well.

With fSession, you can name a session to work with it by chaining or embedding it within another fSession object. The fSession object in addition to keeping track of your sessions also provides a clean and consistent interface for the exposed JavaScript API. For example to originate a call from JavaScript one would normally use the the following commands:


/* session - is a global thing */
var new_session = new Session();
while (new_session.ready()) {
    result = new_session.originate(session, 'sofia/0005551212');
}

This same block of commands could be executed in the fSession one-liner. The session.ready() and new session are automatically created and destroyed for you.

f$('named-session').originate('sofia/0005551212');

A quick example of a fSession time conversion. To quickly convert a time of 5 min and 18 seconds to seconds the following script could be used:

f$.time('5m18s').toSecond();

fSession provides a nice interface to some of the more complex concepts for using javascript, such as the cURL and email. For example to use the cURL functions of javascript, one can just do the following:

f$.post('http://example.com/post.cgi', {'key1':'value1', 'key2':'value2'}/*, optional callback */);

You can also get an optional callback from the posted values, when they come back into JS. It makes it easy for those coming from a web background to get up and running quickly with using JavaScript in FreeSWITCH.

There is a Mock Session object and console_log function wrapping so FreeSWITCH development can be done in a webrowser away from the actual FreeSWITCH machine. This can help somewhat with debugging.
